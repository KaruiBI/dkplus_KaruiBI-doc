# 为什么选择 dkPlus

`dkPlus` 适合做一套更偏业务落地的 Vue 组件方案，重点不是“演示很多炫技 API”，而是帮助项目更快统一界面风格、交互体验和接入方式。

## 它适合的场景

- 中后台管理系统
- 数据分析与 BI 页面
- 需要统一组件风格的企业项目
- 希望在 Vue 3 项目中快速建立交互规范的团队

## 它解决的问题

### 1. 组件接入成本高

很多业务同学卡在第一步，不知道应该装哪个包、引哪个样式、代码放在哪。

### 2. 文档偏开发者，不够产品化

如果文档只是“组件 API 列表”，那用户还是不知道怎么在真实项目里用。

### 3. 官网与项目资料分散

把官网入口、安装说明、快速开始、常见问题统一到一个站点里，用户更容易找到答案。

## 这次搭建的站点目标

这不是单纯复制一份旧文档，而是先把 `dkPlus` 官网最需要的内容骨架搭起来：

- 首页有产品定位
- 文档页有明确阅读路径
- 新手能快速安装
- 后续可以继续补组件说明和案例
