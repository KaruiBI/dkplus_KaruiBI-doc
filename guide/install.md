# 页面指引

这一页的目的，是先把 KaruiBI  的主页面结构讲清楚。  
你可以把它理解成“第一次进入系统时，应该先认识哪些区域”。

## 先看首页工作台

登录成功后，默认进入工作台首页。  
这里不是单纯的欢迎页，而是后续所有功能的总入口。

建议按这个顺序看页面：

1. 左侧主导航
2. 中间项目卡片区
3. 左上角 `新建`
4. 右上角用户与系统设置

## 左侧导航看什么

左侧导航决定了系统能力怎么分区。  
通常可以从这里快速判断当前页面属于哪类功能：

- 项目与工作台
- 模板与组件
- 数据集
- 字典管理
- 用户管理

## 中间区域看什么

中间卡片区通常是“项目入口展示区”。  
如果你是在做培训或交付，这一块最适合先讲：

- 哪些是当前可进入的项目
- 项目缩略图和状态怎么看
- 从哪里进入已有项目

## 右上角看什么

右上角更偏个人设置和系统辅助功能。  
这一块不一定要一开始细讲，但至少要告诉使用者：

- 这里可以切换主题或界面设置
- 这里能看到当前登录账号
- 个人相关操作通常从这里进入

## 页面讲解建议

如果你要向别人介绍首页，不要一开始就讲细节按钮。  
更推荐先讲“页面分区”，再讲“功能入口”，最后再进入具体模块。
